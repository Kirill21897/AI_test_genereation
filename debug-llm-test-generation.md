# Debug Session: llm-test-generation

- Status: OPEN
- Goal: довести проект до полностью корректного рабочего состояния
- Scope: CLI, генерация теста, judge, конфигурация через `.env`, JSON parsing

## Initial Hypotheses

1. Конфигурация провайдера LLM частично некорректна и вызывает нестабильные runtime-ошибки.
2. Схема `GeneratedTest` и фактический JSON-ответ модели не совпадают.
3. В коде есть временные/обходные изменения, которые ломают основной flow.
4. Ошибки связаны не только с кодом, но и с ответами OpenRouter при разных лимитах/балансе.
5. Поток `generate -> parse -> judge -> return` не полностью согласован между файлами.

## Evidence Summary

- `pre-fix`: конфигурация клиента корректно читалась из `.env`, но `moonshotai/kimi-k2.6:free` стабильно падал на `429` upstream rate limit.
- `post-fix`: проект начал читать список моделей из `.env` и пробовать fallback-кандидатов последовательно.
- `post-fix`: был получен реальный JSON-ответ модели, который не соответствовал схеме `GeneratedTest` (`test_title`, `total_questions`, `options` как объекты).
- `post-fix`: подтверждено переключение на второй model id из `.env`.
- Текущий внешний блокер: OpenRouter возвращает `402` из-за недостатка кредитов, когда обе модели недоступны или требуют больше токенов, чем позволяет баланс.

## Fixes Applied

- Добавлено fallback-переключение между несколькими моделями, заданными через `.env`.
- Добавлены timeout и отключение внутренних retry SDK, чтобы не зависать на одном кандидате.
- Усилен prompt-контракт для точной JSON-схемы.
- Добавлена нормализация реального LLM-ответа в формат `GeneratedTest`.
- Обновлён `.env.example` и рабочий `.env` под список кандидатов.

## Status

- OPEN
- Кодовая логика исправлена и локально проверена smoke-тестом нормализации.
- Полный end-to-end прогон сейчас ограничен внешним `402 Payment Required` от OpenRouter.
